💬 常見問題 (FAQ)

      


        

          
❓ Commit 失敗了，是什麼原因？

          
如果看到「工作區很乾淨」就表示沒有任何變更需要提交。你需要先修改檔案，然後 Stage 暫存，再 Commit。如果有變更但還是失敗，可能是忘記 Stage（暫存）了。

        


        

          
❓ Stage All 跟 Unstage 是什麼意思？

          
想像打包行李箱 ✈️：

          
    
            
  • Stage = 把衣服放進行李箱（標記「我要提交這些檔案」）
    • 
            
  • Unstage = 從行李箱拿出來（「先不提交」）
    • 
            
  • Commit = 把行李箱鎖上寄出（正式記錄變更）
    • 
          

          
可以只 Stage 部分檔案，分次提交不同的變更。

        


        

          
❓ 開發中應該持續建立分支嗎？

          
不需要！平常開發直接在 master 上 commit 就好。只有想實驗新功能、不確定會不會成功的時候才建分支。

          
    
            
  • 實驗成功 → 合併回主線
    • 
            
  • 實驗失敗 → 切回主線，刪掉實驗分支
    • 
            
  • 在 master 上改壞 → 用「⏪ 回到此版」回到之前的版本
    • 
          

        


        

          
❓ 可以回到特定分支階段嗎？

          
可以！在 Commit 歷史中：

          
    
            
  • 「🔍 查看」— 先看程式碼變更，確認是哪個版本
    • 
            
  • 「⏪ 回到此版」— 直接回到那個版本（後續變更會丟失）
    • 
            
  • 「⤴️ 建立分支」— 從那個版本建立新分支，保留原本歷史
    • 
          

        


        

          
❓ 這個平台支援多人開發嗎？

          
這是本機單人的版控工具。多人協作時，每人各自用自己的本機工具，透過 Push/Pull 和遠端同步。

          
推送到遠端的是你本機的 commit 歷史，和別人的可能不同。建議推送前先 Pull 同步。

        


        

          
❓ 如何推送到 GitHub / GitLab / Gitee？

          
    
            
  1. 在平台上建立新的 Repository
    2. 
            
  2. 在本工具的「Remote 管理」點「+ 新增」，填入名稱和 URL
    3. 
            
  3. 選擇 Remote 和分支，點「Push 推送」
    4. 
          

          
首次推送可能需要設定 SSH Key 或 Personal Access Token 驗證身份。

        


        

          
❓ 可以打包成桌面應用嗎？

          
可以！用 Electron 就能將這個工具打包成 .app（Mac）或
            .exe（Windows），雙擊就能用，不需要終端機。但不適合部署到雲端（如 Zeabur），因為需要存取本機檔案。
          

        


      

      

        
      

    

  


  
  

    

      
📖 使用說明

      


        

          
🚀 快速開始

          
    
            
  1. 設定工作目錄 — 輸入或瀏覽你的 Git 專案資料夾
    2. 
            
  2. Stage 檔案 — 點擊「Stage All」或勾選特定檔案
    3. 
            
  3. Commit — 輸入變更說明，點擊「Commit」
    4. 
            
  4. Push — 選擇 Remote 和分支，點擊「Push 推送」
    5. 
          

        


        

          
📦 Stage 與 Unstage 是什麼？

          
想像你在打包行李箱準備出國 ✈️：

          
    
            
  • 修改的檔案 = 散落在房間的衣服（你改了但還沒決定要不要提交）
    • 
            
  • Stage (暫存) = 把衣服放進行李箱（標記「這些檔案我要提交」）
    • 
            
  • Unstage = 把衣服從行李箱拿出來（「這個先不提交」）
    • 
            
  • Commit = 把行李箱鎖上寄出（正式記錄這次變更）
    • 
          

          
💡 實用場景：假設你改了 10 個檔案，但只想先提交 3 個，就可以只 Stage 那 3 個再 Commit，剩下的之後再處理。

        


        

          
🌿 分支策略建議

          
平常開發不需要建分支，直接在主線上 commit 就好：

          

            master ── commit 1 ── commit 2 ── commit 3 ── ...
          

          
什麼時候才需要分支？

          
    
            
  • 想試驗新功能，不確定會不會成功 → 建立分支實驗
    • 
            
  • 實驗失敗 → 切回主線，刪掉實驗分支
    • 
            
  • 實驗成功 → 合併回主線
    • 
            
  • 改壞了想完全回到之前 → 用「回到此版」
    • 
          

        


        

          
⏪ 回到特定版本

          
在「Commit 歷史」中，每個 commit 右側有三個按鈕：

          
    
            
  • 「🔍 查看」 — 查看該 commit 的程式碼變更內容（diff）
    • 
            
  • 「⏪ 回到此版」 — 將工作目錄重置到該 commit（警告：後續變更會丟失）
    • 
            
  • 「⤴️ 建立分支」 — 從該 commit 建立新分支，保留原本歷史
    • 
          

        


        

          
👥 關於多人協作

          
這個工具是本機單人使用的版控工具。多人協作時：

          
    
            
  • 每人在自己的電腦上有各自的本機副本
    • 
            
  • 透過 Push 把自己的變更推送到遠端（GitHub 等）
    • 
            
  • 透過 Pull 把別人的變更同步到本機
    • 
            
  • 如果兩人改了同一個檔案，可能會產生衝突 (conflict)，需要手動解決
    • 
          

          
💡 建議：推送前先 Pull 同步一下，確保本機是最新狀態再 Push。

        


        

          
⌨️ 鍵盤快捷鍵

          

            
Enter 設定工作目錄 / 確認 Remote

            
Ctrl/⌘ + Enter 快速 Commit

          

        


        

          
🔗 如何推送到 GitHub / GitLab / Gitee

          

            

              
1

              

                建立遠端 Repo
                
前往平台建立一個新的 Repository：

                
    
                  
  • GitHub → github.com/new
    • 
                  
  • GitLab → gitlab.com/projects/new
                  
    • 
                  
  • Gitee → gitee.com/projects/new
    • 
                

              

            

            

              
2

              

                新增 Remote
                
在本平台的「Remote 管理」區塊，點擊「+ 新增」：

                
    
                  
  • 名稱：github、gitlab、gitee
    • 
                  
  • URL：從平台複製的 repo URL（例如 https://github.com/user/repo.git）
    • 
                

              

            

            

              
3

              

                推送
                
選擇 Remote 和分支，點擊「Push 推送」。或點「推送到所有 Remote」一次同步！

              

            

          

        


        

          
⚠️ 首次推送時，平台可能會要求驗證身份。請確保已設定 SSH Key 或 Personal Access Token。
